Servidores Web de Altas Prestaciones

Tema 1. Introducción


Ejercicio

Buscar información sobre las tareas o servicios web para los que se usan más los programas que comentamos al principio de la sesión:


apache

Desde el 96 ha sido el servidor más popular por lo que cuenta con extensa documentación y una gran comunidad de usuarios.

Su gran inconveniente es que maneja el tráfico web a través de hilos y esto, cuando hay muchas peticiones, puede tener un impacto negativo en el rendimiento causado por un gran número de procesos en memoria. 

Se utiliza para manejar la mayoría de páginas web con un tráfico medio-bajo. Ejemplo: Hosting web de bajo coste.


nginx

Su popularidad ha crecido debido a que utiliza pocos recursos y ofrece una gran escalabilidad incluso en hardware de bajas prestaciones. 

Cada proceso de nginx es capaz de manejar miles de conexiones lo que se traduce en un uso de CPU que se mantiene estable aunque la carga de peticiones sea alta. 

Se utiliza en sitios cuyo tráfico es alto y que se actualizan constantemente. Ejemplo: Red social.


thttpd

Esta aplicación ofrece lo necesario para correr como un servidor HTTP. Esto hace que sea ligero, rápido, portátil y seguro.

Como consecuencia de lo anterior, no vamos a poder obtener todas las funcionalidades de otros servidores web, como por ejemplo, apache.

Se utiliza en máquinas con un hardware antiguo, con necesidades muy básicas o cuya carga de tráfico sea muy alta.


Cherokee

Servidor web rápido, multiplataforma y fácil de configurar.

Soporta diversas tecnologías como FastCGI, SCGI, o Reverse HTTP Proxy.

Se utiliza en servidores que necesiten la mayoría de funcionalidades que ofrece apache pero quieran correr una aplicación más ligera y que aporte mayor rendimiento.


node.js

Es un entorno de ejecución de E/S dirigido por eventos y asíncrono que se ejecuta sobre JavaScript.

Aporta un gran rendimiento debido a su Bucle de Eventos que se encarga de gestionar las operaciones asíncronas y permite escalar grandes volúmenes de clientes.

Esta aplicación es ideal para aquellos servidores en los que el número de conexiones concurrentes sea muy alto.
